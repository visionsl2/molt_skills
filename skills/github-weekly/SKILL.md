# GitHub 热门项目周报

## 功能
- 搜索GitHub上按总星标数排序的热门项目
- **自动去重**：检查memory中已推荐的项目，避免重复推荐
- 整理项目信息并发送给用户

## 操作流程

### 1. 检查去重记录
**必须先执行此步骤！**

使用 `memory_recall` 工具查询已推荐的GitHub项目：
- 查询关键词："GitHub热门项目周报" 或 "github-weekly"

如果找到记录，提取已推荐的项目的完整名称（如 `owner/repo` 格式）。

### 2. 搜索GitHub热门项目
使用浏览器打开GitHub搜索页面：
```
https://github.com/search?q=stars%3A%3E100000&type=repositories&sort=stars
```

获取按星标降序排列的项目列表。

### 3. 过滤已推荐项目
从搜索结果中**排除**已推荐过的项目，找到第一个**新的**项目。

### 4. 整理项目信息
获取以下信息：
- **项目名称**（如：owner/repo）
- **项目链接**（https://github.com/owner/repo）
- **简介描述**
- **星标数、Fork数**等关键数据
- **主要用途**

### 5. 发送用户
使用飞书消息发送给用户（使用当前会话的 message 工具）

**推荐格式：**
```
📊 GitHub热门项目周报

🏆 排名第X的热门项目

---

### 📌 项目名称
[**owner/repo**](链接)

---

### 📝 项目简介
> 项目描述

---

### ⭐ 项目亮点
| 指标 | 数据 |
|------|------|
| **总星标数** | X万⭐ |
| **Fork数** | X千 |

---

### 🎯 用途
用途说明

---
```

### 6. 更新去重记录
**必须执行！**

使用 `memory_store` 工具更新已推荐项目记录：
- **text**: "GitHub热门项目周报去重记录：已推荐项目列表（按推荐时间倒序）。每次搜索前必须检查此列表，跳过已推荐的项目。\n\n已推荐项目：\n- YYYY-MM-DD: owner/repo (星标数)\n- ..."
- **category**: "preference"
- **importance**: 0.9

### 7. 保存报告（可选）
- 保存到 ~/openclaw_doc/reports/github_hot_YYYYMMDD.md

## 注意事项
- **去重是必须的**，每次都必须先检查memory中的记录
- 如果所有高星项目都已推荐过，可以降低搜索阈值（如 stars:>50000）
- 优先推荐实用性强的项目，而非纯技术Demo
