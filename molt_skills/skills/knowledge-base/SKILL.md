# knowledge-base Skill

> 知识库Skill - 向量语义检索 + Word图片提取
> **版本: 1.1.0** (2026-03-11)

## 触发条件
用户发送文件、URL或直接提问时，自动进行知识库检索

## 功能列表

### 1️⃣ 自动检索知识库
- 用户提问时，先从本地知识库语义搜索相关内容
- 基于向量相似度检索，理解语义而非关键词匹配
- 搜不到相关内容时，自动fallback到网上搜索

### 2️⃣ 添加内容到知识库
- 用户发送文件时，自动提取内容并向量化存储
- 支持多种文件格式自动解析
- **重要：记录文件名到MEMORY.md**

### 3️⃣ Word图片提取
- 自动从Word文档中提取图片并保存
- 检索结果返回文字 + 相关图片路径

---

## 文件管理规则（重要！）

**用户发送文件给你时，必须：**
1. 将文件添加到知识库（向量化存储）
2. 记录到 MEMORY.md 的知识库清单（文件名、日期、说明）
3. 告知用户已添加

**知识库清单格式：**
```markdown
| 日期 | 文件名 | 说明 |
|------|--------|------|
| 2026-03-11 | 客户简报.docx | 广州市水务集团客户信息 |
```

---

## 支持的文件格式

| 类型 | 状态 | 说明 |
|------|------|------|
| txt/md | ✅ | 直接读取 |
| docx | ✅ | Word文档 + **图片提取** |
| pdf | ✅ | PDF文档 |
| xlsx/xls | ✅ | Excel表格 |
| url | ✅ | 网页抓取 |

---

## 工作流程

```
用户发送文件 → 内容提取 → 向量化存储
               ↓
          记录到MEMORY.md知识库清单
                              ↓
用户提问 → 语义检索 → 返回结果
```

---

## 图片存储说明
- 目录: ~/openclaw_doc/knowledge/images/
- 格式: {doc_id}_{序号}.{png|jpg|jpeg}
- 检索时可返回相关图片路径

---

## 技术栈
- 向量数据库: LanceDB
- Embedding模型: Ollama nomic-embed-text (768维)
- 文件解析: pypdf, openpyxl, python-docx

---

## 使用示例

**添加文档：**
- 用户发送PDF/Word文件 → 自动提取文字并存入知识库
- 自动记录文件名到MEMORY.md
- 告知用户已添加

**检索知识：**
- 用户提问 → 先搜本地知识库 → 返回结果

**检索结果包含：**
- 标题
- 内容摘要
- 原文（长文本）
- 相关图片列表
